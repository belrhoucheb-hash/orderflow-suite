# Management Systeem

## Kernprincipes

1. **Workers produceren, managers filteren, CEO beslist**
   - Workers leveren gestructureerde output, nooit vrije tekst
   - Managers bundelen, prioriteren en filteren
   - CEO krijgt alleen executive-level informatie

2. **Geen agent-theater**
   - Elke agent heeft een scherp afgebakende rol
   - Geen overlap in verantwoordelijkheden
   - Agents doen alleen wat ze moeten doen

3. **Continue verbeterlus**
   - Analyseren → signaleren → voorstellen → prioriteren → bouwen → reviewen → leren

4. **Mens-in-de-loop altijd**
   - Zie `human-in-the-loop-policy.md` voor details
   - Bij twijfel: altijd escaleren

## Besluitvormingsmodel

### Niveau 1: Worker autonoom
- Analyses uitvoeren
- Bevindingen documenteren
- Standaard-output genereren

### Niveau 2: Manager autonoom
- Lage-impact items goedkeuren/afwijzen
- Backlog herprioriteren (minor)
- Workers bijsturen
- Interne coördinatie

### Niveau 3: CEO beslissing nodig
- Nieuwe features (scope > S)
- Architectuurwijzigingen
- Prioriteitswijzigingen op roadmap
- Wijzigingen in mens-in-de-loop beleid
- Budget/resource beslissingen
- Productie-impactvolle wijzigingen

## Kwaliteitsstandaarden

### Worker output
- Altijd gestructureerd format (zie agent-bestanden)
- Altijd confidence-niveau
- Altijd concrete bestandslocaties
- Altijd actionable aanbevelingen

### Manager output
- Altijd geprioriteerd
- Altijd met aanbeveling
- Altijd impact-beoordeling
- Overlap gefilterd

### CEO brief
- Leesbaar in 2-3 minuten
- Geen jargon
- Concrete aanbevelingen bij beslispunten
- KPI's met trend-richting
