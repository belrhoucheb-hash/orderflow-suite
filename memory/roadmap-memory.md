# Roadmap Memory

## Visie
Een slim, open TMS-systeem dat 90-95% automatisch werkt met menselijke controle op kritieke punten.

## Huidige modules en status

| Module | Status | Automatisering | Prioriteit |
|--------|--------|---------------|------------|
| Inbox/Email Parsing | Gebouwd (basis) | ~40% | Hoog |
| Orders (CRUD) | Gebouwd | ~50% | Hoog |
| AI Extractie (parse-order) | Gebouwd (basis) | ~30% | Hoog |
| AI Corrections | Schema klaar | ~10% | Hoog |
| Planning/VRP | Gebouwd (basis) | ~30% | Middel |
| Dispatch to Delivery | Schema klaar (trips, stops, POD) | ~5% | Hoog |
| Chauffeur App | Gebouwd (basis) | ~40% | Middel |
| Fleet Management | Gebouwd (basis) | ~30% | Laag |
| Facturatie | Gebouwd (basis) | ~20% | Middel |
| Track & Trace | Gebouwd (basis) | ~20% | Laag |
| Dashboard/KPI's | Gebouwd (basis) | n/a | Laag |
| Multi-tenant | Gebouwd | n/a | Onderhoud |

## Strategische prioriteiten

1. **Email → Order pipeline versterken** — dit is de kern van automatisering
2. **Dispatch to Delivery bouwen** — schema is klaar, UI moet volgen
3. **AI Corrections feedback loop** — leren van fouten
4. **Planning optimalisatie** — VRP solver uitbreiden
5. **Facturatie automatiseren** — van aflevering naar factuur
